# 2026/02/19 - Seed7

## なぜ Seed7 はZigと拮抗できるくらい優れているのに普及の兆しを見せず誰もを使おうとしないのか？

* **DOOMが移植されていない**: マイナー言語にとって DOOM の完全移植はメジャー言語への登竜門であり、その他多くマイナー言語とそうでない言語をハッキリ分ける残酷な分水嶺である。普通の人にとってなんかまたDOOMの移植増えている程度の認識だし、一部の言語開発者にとってDOOMとはベンチマークや基礎教養であることは公然の暗黙知です。できるだけCルーチン (FFI, トランスパイラ, インライン) に頼らず言語の機能を活用した上でDOOMが移植されていない言語は使い物にならないのは暗黙の了解。まあ、そんなの誰も言わないし言われなきゃわからんよな。ウチも言われなければ気づかんかったよ。
* ドキュメントシステムの問題: HTML 3.2 / 4.0 形式はlynx, w3m があれば読めるんでポータビリティに優れているがデカすぎて滅茶苦茶翻訳しづらいし機械翻訳でスタイルや訳出破綻するので非英語圏では広まりづらい。mkdocなど現代的なドキュメントシステムを導入することで翻訳やら編集管理が楽になると。探せば外部依存性のないシングルバイナリのドキュメントジェネレーターもあるんよね。
* シンタックスの心理的障壁: きちんとした理由があってこうなっているのはわかるんだけど現代の抽象化構文(スマートフォン)慣れたプログラマにとって過剰な具象化構文(一時期の国内メーカー製の白物家電みたいなUX)ショック死しかねない。まあ、構文拡張機能があるからといって魔改造してまで使いたがる変態σ(ﾟ∀ﾟ )ｵﾚ?は少ない。理由は魔改造すると Seed7 のバージョンアップのたびに修正が必要になるとか、その言語拡張機能を使って書いたソフトウェアは他人に読めないコードになって保守性が落ちるからです。 Silver や Ring が敬遠されているのと同じ理由ですね。

トーマス・メドレーさん作者本人が有能すぎて周りがついていけなかったり天上天下唯我独尊過ぎるなどその他色々と考えられますけど、いまは仕方ないかな。ちゃんとした経験豊富な言語デザイナーやらUXデザイナーをプロジェクトに招き入れるだけで変わるかもしれないですね。
