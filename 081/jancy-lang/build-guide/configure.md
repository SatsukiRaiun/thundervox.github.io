# 設定・構成〔コンフィギュレーション〕

さて、必要なツールとライブラリはすべてあなたのマシンで利用できます。次は何ですか？　すべての CMake ベースのプロジェクトと同様に、ビルドを設定し、適切な makefile を作成する必要があります。

しかし、ほとんどのCMakeベースのプロジェクトとは重要な違いがあります。 Jancy のビルドシステムは、依存関係を見つけるのに CMake の `find_package` 機能に依存していません。なぜいけないのか？

まず、 `find_package` は Windows 上では確実に動作しません。ここでは、パス設定変数の追加レイヤーを設定し、これらのパスを `find_package` のヒントとして提供する必要があります。これにより、 `find_package` の背後にあるアイデア全体が事実上**無効**になります。実際の「検索」を行うのは、 `find_package` ではなくあなたです！

これとは別に重要な理由は、同じライブラリの異なる**バージョン**間、 **Debug/Release** ビルド間**、静的ライブラリ・動的ライブラリ**間、クロスコンパイル中の異なる**プロセッサアーキテクチャ**間などを**切り替える**必要がある場合に、 `find_package` を使用するのは便利ではないということです。

言い換えれば、インストールされているツールやライブラリの**デフォルト**バージョンで**ネイティブ**ビルドを行うだけの場合、 **Unix** システムでは `find_package` が非常に便利です。もっと柔軟性が必要ですか？　それでは、 `find_package` だけでは解決できません。
