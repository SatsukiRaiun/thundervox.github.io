# サンプル

Jancyのビルドとテストが成功したら、 `./samples` フォルダをチェックしてみてください。

 * `jnc`\
   このサブフォルダには、最も顕著な**言語機能**を示す複数の Jancy ソースファイルが含まれています。


 * `jnc_sample_01_embed_c`\
   このサンプルでは、Jancyをスクリプトエンジンとして純粋なCプログラムに埋め込む方法を示します。また、JancyクラスをCで実装する方法と、それらをスクリプトの名前空間にエクスポートする方法も示します。これにより、スクリプトとホストアプリケーション間の双方向の対話が可能になります。\
   \
   以下のように実行します。

 ``` bash
 jnc_sample_01_embed_c <スクリプトのパス>
```
   \
   サンプルスクリプトは、 `./samples/jnc_sample_01_embed_c/script.jnc` にあります。



 * `jnc_sample_02_embed_cpp`\
   このサンプルは、前のサンプルの C++ 版です。**スクリプトエンジン**として Jancy をC ++ プログラムに埋め込む方法と、**Jancy クラスを C++** に実装する方法を示します。\
   \
   以下のように実行します。

 ``` bash
 jnc_sample_02_embed_cpp <スクリプトのパス>
```
   \
   サンプルスクリプトは、 `./samples/jnc_sample_02_embed_cpp/script.jnc` にあります。

 * `jnc_sample_03_dialog`\
   このサンプルは、 Qt ウィジェットクラスへの Jancy バインディングを作成する方法と、 Jancy リアクティブプログラミングの概念をUIプログラミングに適用する方法を示しています。\
   \
   明らかに、このサンプルではQTをビルドして実行する必要があります。\
   \
   以下のように実行します。

 ``` bash
 jnc_sample_03_dialog <スクリプトのパス>
```
   \
   サンプルスクリプトは、 `./samples/jnc_sample_03_dialog/script.jnc` にあります。

また、ユーザコードをコンパイルし、実行することができる `./test/test_qt-Jancy` エディタをチェックすることもできます。

この Qt アプリケーションは、 Jancy コンパイラ、 Jancy ランタイム、および Jancy 標準ライブラリのトラブルシューティングとバグ修正のために常に使用されています。このアプリケーションを前述の目的のために本当に便利にしているのは、_エディタ_、_コンパイラ_、_ランタイム_ のすべてが単一のプロセスの範囲内でブレンドされているため、すべてが可能な限り _デバッガフレンドリー_ になります。
